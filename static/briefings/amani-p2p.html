The Systemic Crisis

This section illustrates the current vulnerabilities in the Kenyan crypto ecosystem. Centralized Exchanges (CEXs) are arbitrarily freezing retail P2P accounts to comply with state mandates, trapping user funds. The radar chart below compares the systemic risk profile of the traditional centralized model against the proposed Amani Protocol framework.

⚠️

Arbitrary Asset Seizure

CEXs hold escrowed funds centrally. Upon law enforcement requests (DCI/ATPU), they instantly trap user funds without due process or formal court orders to protect their own licenses.

👁️

Privacy Friction

Draft VASP Regulations 2026 mandate zero anonymity. Users surrender sensitive PII (KRA PINs, National IDs) to foreign exchanges, creating massive data breach targets.

⏳

Single Point of Failure

Dispute resolutions rely entirely on centralized, often unresponsive support desks, creating massive financial bottlenecks for local traders.

The Amani Solution

Amani P2P shifts trust from corporate entities to immutable code. This section details the three core pillars of the protocol, explaining how compliance is proven mathematically rather than managed manually, thereby protecting users from arbitrary seizures.

🔐

Non-Custodial Escrow

Assets are locked in an immutable EVM smart contract. Only private key holders (Buyer, Seller, Arbitrator) can trigger release. The state cannot freeze the contract via email.

🎭

zk-KYC Identity

Users verify identity once with a regulated oracle to receive a Soulbound Token (SBT). Proves compliance ("Not on AML list") without revealing actual names or IDs on-chain.

⚡

API-Verified Settlement

Integration with local fiat rails (M-Pesa Daraja API) via a decentralized oracle automatically verifies fiat settlement, bridging off-chain payments with on-chain escrow.

Architecture & Compliance

This section outlines the technical stack required for the Minimum Viable Product (MVP) and details how this decentralized architecture fundamentally beats the restrictive Draft VASP Regulations 2026 by complying with the law's intent without sacrificing user sovereignty.

Smart Contracts

⚙️

Solidity Engine

Deployed on low-fee L2s (Base, Arbitrum, Polygon). Handles Create, Lock, Release, and Dispute.

Identity Layer

🆔

Polygon ID / zCloak

Zero-Knowledge layer for issuing and verifying verifiable human credentials required for interaction.

Frontend & Web3

🖥️

Next.js + viem/wagmi

React UI styled with Tailwind, authenticated via WalletConnect/MetaMask.

Infrastructure

🗄️

Off-Chain Relayer & Multi-Sig

Node.js/PostgreSQL order book (0x style). 2-of-3 multi-sig for community-vetted arbitration fallback.

1

No Anonymity (VASP Compliant)

To interact with the Amani contract, a wallet must hold a valid zk-KYC token. Criminals and unverified entities are mathematically barred from the platform, satisfying strict AML monitoring intents.

2

No Centralized Custody

Because Amani does not hold user funds in corporate wallets, it operates technically outside the purview of standard VASP custodian definitions, pushing liability to edge users and removing the central attack vector for arbitrary seizures.

3

Immutable Auditing

The state and regulators can view the public on-chain ledger to verify that 100% of volume passed exclusively through KYC-verified wallets, achieving compliance transparency without needing a backdoor to a central database.
